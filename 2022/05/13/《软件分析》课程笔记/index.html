Programming Languages

  • Environment

How to ensure the reliability, security and other promises of large-scale and complex programs?

静态分析场景：

  • Program Reliability，空指针引用，内存泄露…
  • Program Security，信息泄露，注入…
  • Compiler Optimization
    • 死代码剔除
    • code motion
    • …
  • Program Understanding，IDE调用，类型定义…

静态分析是什么？

在运行程序前，就已经知道有关程序的行为、特性。

  • 莱斯定理：

递归可枚举语言，它的非平凡（non-trivial）特性是不可判断的

no-trivial properites：

  • interesing properties
  • 与运行时有关的特性

Perfect static analysis：完美的静态分析是不存在的

  • Sound
  • Complete

这里对Sound的理解：包含所有Truth，对应于误报。Complete（准确）：包含于Truth，对应于漏报。

Necessity of Soundness

  • 宁可错杀一千，不可放过一个

一句话概括静态分析：

在保证Sound的情况下，对精度和速度做一个平衡。

如何做静态分析：

  • Abstraction（定义抽象集）

  • Over-approximation

    • Transfer Functions（在抽象集上定义转换操作）

  • Control flows

课时二：IR

这节课老师提到了王垠，之前听说过一些牛事

3-Address Code（3AC）

一种中间表示格式

  • 右侧最多一个操作符，t1 = a + b
  • Address
    • 变量名
    • 常量
    • 编译器生成的临时变量

Java静态分析框架：Soot，Soot’s IR：Jimple

  • JVM扩展

SSA静态单赋值（Option）：每个变量有自己的定义

Why SSA?（优点）

  1. 程序Flow information会间接通过变量关联起来。flow-insensitive
  2. 定义与使用是显示的。

Why not SSA?（缺点）

  1. 会引入太多变量和Φ函数

  2. 转换为机器码时，性能问题。太多的copy操作

Control Flow Analysis

  • 建立控制流图
  • 节点即可以是指令也可以是BB

Basic Blocks（BB）

  • BB定义：最多的连续3AC指令序列集合，满足如下特性

    • 入口：第一条指令
    • 出口：最后一条指令

  • BB建立算法
